# Kimi Code Emergency Fallback v0.1

Status: `fallback_ready_with_codex_review_required`.

Kimi Code is a lightweight emergency fallback runner for public paper-radar operations.

Allowed:

- run safe documented manual commands;
- inspect generated Daily, Weekly, Monthly, source-health, reading queue, and Obsidian artifacts;
- summarize outputs against runbook expectations;
- draft low-risk documentation notes.

Forbidden:

- release ownership;
- git add, git commit, git push, or git tag;
- tag creation, deletion, movement, or recreation;
- production code changes unless explicitly authorized and later reviewed by Codex;
- ranking, source fetcher, taxonomy, query expansion, or negative keyword changes;
- private path access;
- scheduled or background automation;
- anti-bot bypass or access-control evasion;
- unsupported command-success claims.

Codex review is required for any code, test, verifier, source-health classification, runbook, or release-gate change.
