# Radar Recency Freshness Filter v0.1

## Purpose

Daily paper radar output uses freshness as a hard gate before recommendation ranking. A highly relevant but stale paper must not enter the primary today/new section unless it has a reliable in-window date basis.

## Default Freshness Window

The default daily window is run date minus one day through run date, inclusive. For a 2026-07-01 run, the default primary window is 2026-06-30 through 2026-07-01.

## Date Basis

Primary today/new eligibility may use:

- publication_date
- announcement_date
- update_date

Non-primary explicit routing may use:

- first_seen_date
- official_status_change_date
- source_metadata_correction_date
- manually_requested_backfill_date
- explicit freshness_reason

Source publication_date is source metadata and must not be replaced by scrape time or first_seen_date.

## Routing

Items outside the primary window are routed outside primary today/new:

- backfill
- important_older_item
- newly_discovered_but_older
- official_status_changed
- source_metadata_corrected
- manually_requested
- high_priority_security_update
- date_uncertain_todo_verify

Older items must carry a freshness_reason. Missing or ambiguous source dates are TODO_VERIFY and non-primary.

## Venue And CCF Metadata

Venue metadata is represented with:

- venue
- venue_type
- publisher_or_source
- CCF_rank
- venue_status
- venue_expanded_security_crypto_systems_scope
- venue_relevance
- venue_confidence

CCF rank may be A, B, C, N/A, unknown, or TODO_VERIFY. The radar must not fabricate CCF rank. Cybersecurity journal style candidates are represented as TODO_VERIFY unless trusted local metadata later establishes a rank.

## Bilingual Summary Fields

Every selected item should expose:

- abstract_en
- abstract_zh
- conclusion_en
- conclusion_zh

Source-provided English abstract is stored as abstract_en when available. Chinese summaries generated by the local model are marked with `model-generated zh summary:`. Missing abstract or conclusion fields use TODO_VERIFY.

## Recommendation Metadata

Recommendation metadata includes:

- recommendation_level
- recommendation_score
- recommendation_reason

Freshness routing runs before recommendation ranking for the primary today/new section. Non-primary older items may retain relevance scores, but ranking cannot promote them into primary today/new.
