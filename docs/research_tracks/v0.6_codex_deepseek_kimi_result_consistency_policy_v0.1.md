# v0.6 Codex / DeepSeek / Kimi Result Consistency Policy v0.1

Status: `cross_operator_compatibility_ready_with_codex_review_required`.

Cross-operator compatibility means:

- all three operators use the same project path;
- all three avoid private paths;
- all three use the same manual commands when running the public radar;
- generated artifacts are compatible artifacts in the same repository paths;
- all three use the same final operation report template;
- source-health caveats and TODO_VERIFY are preserved;
- DeepSeek-Claude and Kimi Code cannot silently change final release outcome;
- Codex review is required for code, test, verifier, source-health, runbook, or release-gate changes.

The policy preserves manual fallback without turning fallback operators into release owners.
