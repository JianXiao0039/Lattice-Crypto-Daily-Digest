# v0.6 Semantic Scholar Long-Run Policy v0.1

Status: `semantic_scholar_policy_ready`.

The radar must only report `SEMANTIC_SCHOLAR_API_KEY` as present or missing. The key value must never be printed.

Semantic Scholar states:

- `missing_key`;
- `auth_failure`;
- `rate_limited`;
- `timeout`;
- `network_error`;
- `empty_response`;
- `enrichment_skipped`;
- `ok`.

Semantic Scholar enrichment is auxiliary. If it fails, Daily/Weekly/Monthly should continue with `enrichment unavailable` style caveats rather than treating papers as irrelevant.
