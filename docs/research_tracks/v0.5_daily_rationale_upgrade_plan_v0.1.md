# v0.5 Daily Rationale Upgrade Plan v0.1

Status: controlled future integration plan.

Daily digest impact:

1. Keep source ingestion, source health, normalization, ranking, and section placement unchanged.
2. Build rationale after records are ranked and before Markdown/JSON rendering.
3. Add rationale fields as explanatory metadata only.
4. Preserve existing `reason_for_priority` and `why_it_matters` during the first compatibility phase.
5. Render a short Chinese rationale block for A/B records after existing ranking evidence.

Recommended Daily text shape:

- Problem: abstract-derived if available.
- Method: extracted from available abstract/conclusion sentence, otherwise TODO_VERIFY.
- Radar relevance: explicit lattice/PQC anchor.
- Reading action: 精读 / 扫读 / 暂存 / 忽略.
- Caveat: what needs full-paper verification.

Non-goals:

- No ranking change.
- No taxonomy change.
- No fetcher change.
- No source-health change.
- No manual annotation dependency.
- No external LLM call.

Gate:

Daily integration should wait for a controlled patch that verifies JSON compatibility, Markdown snapshot stability, and no change to record ordering.
