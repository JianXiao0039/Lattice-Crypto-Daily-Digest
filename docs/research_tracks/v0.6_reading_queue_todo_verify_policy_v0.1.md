# v0.6 Reading Queue TODO_VERIFY Policy v0.1

Status: `todo_verify_grouping_ready`.

Reading queue records should group TODO_VERIFY items into categories:

- proof / security model;
- parameter claim;
- benchmark / implementation;
- construction details;
- attack model;
- dataset / evaluation;
- relation to standard LWE/RLWE/MLWE/SIS;
- source-health / metadata incompleteness.

These categories help the user decide what to check during full-paper reading. They are not user labels, human-gold labels, or classifier targets.
