# v0.5 Module-SIS and Sanitizable Signature Track Scope v0.1

## Include

- SIS, Module-SIS, Ring-SIS, or lattice-trapdoor anchored chameleon hashing;
- sanitizable or redactable signatures with a lattice/PQC construction;
- lattice commitments and trapdoor collision/adaptation mechanisms;
- exposure, accountability, collision resistance, correctness, and adaptation properties;
- reproducible parameterization and implementation;
- comparison with classic chameleon hash when technically relevant.

## Exclude

- generic hash or sanitizable signature work without lattice/PQC anchors;
- generic blockchain redaction;
- claims that SIS and LWE are interchangeable;
- constructions with no explicit trapdoor/collision mechanism evidence.

## Evaluation Questions

- Which exact assumption supports the construction?
- How does the trapdoor enable authorized collision/adaptation?
- Which security property is claimed, and is it verified?
- Can the paper support the quick-paper related work, proof sketch, parameters, or implementation?
