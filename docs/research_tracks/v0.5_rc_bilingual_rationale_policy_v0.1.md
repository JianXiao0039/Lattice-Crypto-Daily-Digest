# v0.5 RC Bilingual Rationale Policy v0.1

Status: `bilingual_rationale_policy_ready`.

## Policy

Use bilingual rationale where it improves reading decisions without bloating every radar output.

For `A-class`, `Must Read`, `top weekly`, and `top monthly` papers, prefer compact bilingual blocks:

中文：

- 论文大致工作：
- 核心创新点：
- 与本雷达关系：
- 建议：
- TODO_VERIFY：

English:

- Paper work summary:
- Core novelty:
- Radar relevance:
- Recommendation:
- TODO_VERIFY:

For B/C peripheral papers, compact Chinese-only output is acceptable.

## Evidence Guardrail

- Use conclusion-derived claims only when a conclusion field exists.
- Use abstract-derived claims only when an abstract exists.
- Title-only or metadata-only records must be marked low-confidence.
- Keyword hits may support rationale but must not be the whole rationale.
