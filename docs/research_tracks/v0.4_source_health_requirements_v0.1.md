# v0.4 Source Health Requirements v0.1

## Per-Source Requirements

- arxiv: classify rate limit, timeout, and generic URL errors.
- crossref: classify network/API failures separately from no matching records.
- dblp: preserve SSL/URL retryable warnings.
- iacr_eprint: distinguish `failed/0`, `fetched`, `cache_hit`, parser failure, and guard behavior.
- openalex: distinguish no records from source failure.
- semantic_scholar: report key presence safely; never print key values.

## Global Requirements

- Preserve green/yellow/red semantics.
- Do not stop the whole workflow solely because optional sources are degraded.
- Do not claim no papers exist when all sources are red.
