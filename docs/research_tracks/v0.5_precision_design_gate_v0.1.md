# v0.5 Precision Design Gate v0.1

## Decision

`design_ready_with_more_annotation`

## Rationale

- A 15-record offline sample covers the required major tracks, ambiguous cases, and false positives.
- Existing repository records provide enough evidence to design explanations, review criteria, and test cases.
- Threshold-PQC-specific coverage is not verified.
- Several construction and AI-method claims still require original-paper verification.

## Allowed

- offline annotation;
- sample expansion and adjudication;
- false-positive/false-negative review;
- track explanation and handoff usefulness design;
- focused test planning.

## Not Allowed by This Gate

- production taxonomy, ranking, query, negative-keyword, or classifier changes;
- automatic ResearchArtifacts synchronization;
- claims of measured production precision.
