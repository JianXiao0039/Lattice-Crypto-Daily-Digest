# v0.5 Final Post-Tag Smoke Plan v0.1

## Purpose

This is a later-phase smoke plan. It is not executed in Phase 13S.

## Smoke Checks After a Future User-Confirmed Tag

1. Verify tag exists locally and remotely.
2. Verify tag points to the intended release commit.
3. Verify GitHub Actions for the tag is green.
4. Run `python -m lattice_digest.workflow doctor`.
5. Verify Daily/Weekly/Monthly artifact commands remain manual-only.
6. Verify source-health probe remains low-load and compliant.
7. Verify reading queue and Obsidian exports still write only inside the repository.
8. Verify release notes do not overstate CI, source health, durable evidence, or v0.4.1 status.

## Boundaries

Do not create scheduled tasks, background services, watcher processes, external LLM calls, manual annotation workflows, or private-path writes during post-tag smoke validation.
