# v0.6 Kimi Code Route Prompt Improvement v0.1

Status: `kimi_code_fallback_ready`.

Kimi Code prompts should be fully self-contained because Kimi Code must not rely on long-term memory.

Required prompt constraints:

- active project path;
- private paths forbidden;
- only explicit safe commands;
- no git add/commit/push/tag;
- no release ownership;
- no background automation;
- no code modification unless explicitly authorized;
- no unsupported command-success claims;
- Codex review for code changes.

Kimi Code is suitable for lightweight emergency manual reruns, artifact inspection, and report comparison. It is not a release or broad-code-change operator.
