# v0.6 Crossref Long-Run Policy v0.1

Status: `crossref_policy_ready`.

Crossref status must separate:

- healthy response;
- valid empty response;
- query mismatch;
- rate limit or HTTP failure;
- network timeout;
- parser failure.

If polite usage metadata is supported by project configuration, use it truthfully. Do not fake identity or rotate identities.

Crossref degradation should affect reliability language but should not stop the radar by itself.
