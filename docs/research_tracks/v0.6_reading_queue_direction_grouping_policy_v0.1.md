# v0.6 Reading Queue Direction Grouping Policy v0.1

Status: `reading_queue_direction_grouping_ready`.

Reading queue records should expose a normalized `research_direction` in addition to existing sections and tags.

Supported directions:

- LWE / RLWE / MLWE;
- SIS / Module-SIS;
- lattice reduction / BKZ / cryptanalysis;
- ML-KEM / ML-DSA / PQC implementation;
- FHE / CKKS;
- ZK / commitments;
- chameleon hash / signatures / anonymous authentication;
- AI-assisted lattice cryptanalysis / AI4LC;
- adjacent crypto / background.

This field is an explanation and grouping layer only. It does not change ranking, source fetching, taxonomy, query expansion, negative keywords, or paper inclusion.
