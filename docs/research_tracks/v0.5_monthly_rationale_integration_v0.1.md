# v0.5 Monthly Rationale Integration v0.1

Status: `monthly_rationale_integrated_with_limits`.

Monthly synthesis uses:

- `lattice_digest.recommendation_rationale.build_recommendation_rationale`

Rationale appears in:

- monthly JSON `core_papers[].rationale`
- monthly Markdown core-paper blocks
- monthly reading-priority entries

Evidence behavior:

- Abstract-derived rationale when abstract exists.
- Conclusion-derived support only when a conclusion field is present.
- Metadata/title-only records stay low-confidence.
- Keyword hits are supporting evidence only.
- TODO_VERIFY caveats are retained for missing full text, missing conclusion, missing abstract, weak source-health, or weak relevance evidence.

Limit:

The integration does not add rationale fields to Daily or Weekly JSON. Phase 13J's Markdown-only Daily/Weekly policy remains unchanged.
