# v0.4 Source Reliability Regression Tests v0.1

## Objective

Protect deterministic backfill, source-health visibility, and source-starved artifact generation without changing ranking, taxonomy, query expansion, or source selection semantics.

## Required Regression Cases

| Case | Expected result |
|---|---|
| Valid `--date YYYY-MM-DD` | accepted and parsed as one date |
| Invalid date | clear argparse failure |
| `--date` with `--since` | rejected |
| Existing `--since` | behavior remains available |
| Exact-day coverage | only records in the Asia/Singapore day remain |
| Zero records | JSON and Markdown still exist |
| All sources red | source health remains red/retryable and artifact remains readable |
| IACR cache hit | status and record count remain visible |
| Semantic Scholar unavailable | enrichment limitation is visible without exposing the key |
| Private path guard | tests write only to temporary repository roots |

## Non-Regression Boundaries

- No ranking threshold changes.
- No taxonomy or classifier changes.
- No fetcher or source-selection changes.
- No automatic retries.
- No private workspace writes.
- No Git mutation operations.
