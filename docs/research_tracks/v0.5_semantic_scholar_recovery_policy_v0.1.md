# v0.5 Semantic Scholar Recovery Policy v0.1

Status: `source_health_diagnostics_ready`.

Semantic Scholar is optional enrichment. Its failure must not make papers irrelevant.

## Required Behavior

- Report API key presence as a boolean only.
- Never print the API key.
- Separate missing key, authentication failure, rate limit, timeout, network failure, empty response, and no-candidate enrichment.
- Respect rate limits.
- Report enrichment unavailable when appropriate.

## Phase 13L Result

The low-load probe reported Semantic Scholar key presence as `true` without exposing the key. The endpoint returned HTTP 200 and one candidate.
