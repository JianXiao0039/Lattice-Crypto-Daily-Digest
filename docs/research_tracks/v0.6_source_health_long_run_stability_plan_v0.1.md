# v0.6 Source Health Long-Run Stability Plan v0.1

Status: `source_health_long_run_policy_ready`.

Long-run stability means the radar remains honest and usable when sources degrade.

The plan is:

1. keep source probes manual and low-load;
2. classify failures by cause rather than generic failure;
3. surface source-starved days explicitly;
4. avoid treating failed sources as no relevant papers;
5. preserve Daily/Weekly/Monthly output generation when enrichment sources fail;
6. use manual retry later instead of background retry loops;
7. keep anti-abuse compliance visible in operator reports.

This plan does not change ranking, thresholds, taxonomy, query expansion, negative keywords, or source fetcher behavior.
