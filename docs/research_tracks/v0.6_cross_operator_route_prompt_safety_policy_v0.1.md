# v0.6 Cross-Operator Route Prompt Safety Policy v0.1

Status: `codex_deepseek_kimi_prompt_safety_ready`.

Every route prompt must clearly define:

- project root: `D:\Code\CodexProjects\lattice-crypto-daily-digest`;
- private path freeze: `D:\Code\CodexProjects\PhD_Application`, `D:\ResearchArtifacts`, and `D:\ResearchOS`;
- no git add/commit/push/tag unless explicitly authorized in a separate phase;
- no background automation;
- no source/ranking/taxonomy changes unless the phase explicitly allows them;
- no external LLM runtime;
- no manual annotation workflow;
- required pre-run checks;
- required source-health caveats;
- required final report;
- required Codex review for code changes.

A route prompt is safe only if Codex, DeepSeek-Claude, and Kimi Code can run it with compatible boundaries and outputs, while DeepSeek-Claude and Kimi Code remain fallback runners with no release ownership.
