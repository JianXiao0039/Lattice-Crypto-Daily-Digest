# v0.5 Monthly Bilingual Rationale Case Review v0.1

## Decision

`bilingual_policy_documented_but_not_rendered`

## Evidence

The policy file `docs/research_tracks/v0.5_rc_bilingual_rationale_policy_v0.1.md` requires or recommends compact bilingual rationale for A-class, Must Read, top weekly, and top monthly papers.

The real Monthly artifact `digests/monthly/2026-06.md` includes strong structured English-labeled fields:

- `Problem`
- `Method / construction / attack / implementation`
- `Contribution`
- `Radar relevance`
- `Evidence basis`
- `TODO_VERIFY`

But it does not render a separate compact `English:` block for top papers.

## Interpretation

This is not a keyword-only failure. It is a presentation gap against the preferred bilingual policy.

## Recommended Follow-Up

In a later rendering-only phase, add a compact bilingual top-paper block or explicitly document that Monthly uses English field labels with Chinese recommendation text as the compact alternative.
