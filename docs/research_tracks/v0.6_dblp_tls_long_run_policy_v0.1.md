# v0.6 DBLP TLS Long-Run Policy v0.1

Status: `dblp_tls_policy_ready`.

DBLP failures must distinguish:

- SSL certificate validation failure;
- TLS handshake failure;
- DNS/network failure;
- timeout;
- HTTP failure;
- parser failure;
- valid empty response.

SSL verification must not be disabled as a production default.

If TLS fails, document local CA/proxy/network remediation and retry manually later. Do not convert TLS failures into no-paper claims.
