# v0.6 arXiv Rate-Limit Long-Run Policy v0.1

Status: `arxiv_rate_limit_policy_ready`.

HTTP 429 must be classified as `rate_limited`, not generic failure.

Operator policy:

- honor `Retry-After` when visible;
- use low-load rerun later;
- avoid tight retry loops;
- reduce query volume if needed;
- mark source-starved output if arXiv materially failed;
- do not claim no relevant papers when arXiv failed.

Forbidden:

- proxy rotation;
- fake User-Agent rotation;
- hidden browser automation;
- ignoring rate limits or source terms.
