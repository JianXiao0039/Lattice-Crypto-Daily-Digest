# v0.5 Post-Tag Observation Plan v0.1

## Daily

- Record workflow/run ID, start time, commit, artifact date, and persistence status.
- Confirm Markdown and JSON exist in `origin/main`.
- Record source green/yellow/red counts, source-starved status, IACR latest, and Semantic Scholar enrichment state.
- Treat generation followed by commit failure as an automation attempt, not a successful persisted run.

## Weekly

- Record the named Weekly automation run ID or equivalent authoritative evidence.
- Verify input dates, source-starved propagation, and handoff packet traceability.
- Keep manual handoff replay separate from actual Weekly automation evidence.

## CI

- Track Windows and Ubuntu separately.
- Inspect authenticated failed logs when available.
- Do not infer a root cause from the red status alone.

## Minimum Next Evidence

- multiple persisted post-tag Daily runs;
- one persisted post-tag Weekly run;
- at least one candidate-bearing Semantic Scholar observation;
- IACR latest observations under normal and degraded conditions;
- no silent artifact omissions.
