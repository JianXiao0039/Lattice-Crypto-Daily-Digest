# v0.5 Reading Queue Policy v0.1

Status: `reading_queue_polished`.

The reading queue is a local usability layer over existing Daily / Weekly / Monthly radar artifacts. It does not fetch papers, rerank papers, change taxonomy, or require user annotation.

## Record Fields

Reading queue records may include:

- `paper_id`
- `title`
- `authors`
- `source`
- `source_url`
- `doi`
- `arxiv_id`
- `eprint_id`
- `date`
- `score`
- `class_label`
- `reading_priority_score`
- `reading_action`
- `direction_tags`
- `radar_track`
- `rationale_problem`
- `rationale_method`
- `rationale_contribution`
- `rationale_relevance`
- `rationale_caveat`
- `evidence_basis`
- `source_health_context`
- `TODO_VERIFY`
- `first_seen`
- `last_seen`
- `seen_in_daily`
- `seen_in_weekly`
- `seen_in_monthly`

Allowed local workflow fields:

- `reading_status`
- `review_status`
- `user_notes`, if added later as an optional blank field.

Forbidden annotation fields:

- `user_label`
- `human_gold_label`
- `user_confirmed`
- `user_corrected`
- `manual_annotation_status`

## Noninterference

Reading action is an explanation layer. It must not change ranking score, relevance label, source ingestion, Daily output inclusion, Weekly synthesis inclusion, or Monthly synthesis inclusion.
