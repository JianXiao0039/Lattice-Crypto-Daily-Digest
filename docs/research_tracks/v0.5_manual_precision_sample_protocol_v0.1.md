# v0.5 Manual Precision Sample Protocol v0.1

## Purpose

Build an offline, reviewable annotation sample from existing repository records without changing production classification.

## Selection

Include positive, negative, and ambiguous records covering:

- Module-SIS, commitments, trapdoors, chameleon hashing, and sanitizable signatures;
- lattice ring/linkable/blind signatures and Xingye Lu bridge topics;
- ML-KEM, ML-DSA, threshold-PQC background;
- LWE/RLWE/MLWE attacks and lattice reduction;
- AI-assisted or potentially learning-guided lattice attacks;
- obvious false positives and hard ambiguous items.

## Evidence Rules

- Use only repository identifiers, titles, sources, and metadata.
- Do not infer paper claims from keywords alone.
- Mark missing assumptions, methods, authorship, and threshold coverage `TODO_VERIFY`.
- Keep ambiguous examples rather than forcing a binary label.
- Never claim professor-specific authorship or fit from this sample.

## Review Process

1. Generate the offline sample.
2. Manually verify identifiers and original sources.
3. Review false positives and likely false negatives.
4. Record primary/secondary track disagreements.
5. Approve a gold sample before proposing production changes.

Current generated sample size: 15 records.
