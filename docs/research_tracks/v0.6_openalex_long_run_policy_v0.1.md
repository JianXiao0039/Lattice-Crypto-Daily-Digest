# v0.6 OpenAlex Long-Run Policy v0.1

Status: `openalex_policy_ready`.

OpenAlex must distinguish a valid empty response from a network failure.

Operator checks:

- verify query and date-filter assumptions;
- avoid overinterpreting zero results;
- report source-health caveat when network or parser failure occurs;
- retry manually later in low-load mode if degraded.

OpenAlex zero results are not by themselves evidence that the month or day had no relevant papers.
