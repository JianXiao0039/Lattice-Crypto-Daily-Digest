# v0.5 Rationale JSON Schema Policy v0.1

Phase 13J decision:

Do not change Daily or Weekly JSON schema yet.

Reason:

- Markdown rationale can improve readability without risking downstream JSON consumers.
- Existing JSON tests and generated artifacts remain compatible.
- A future controlled patch may add optional rationale fields after schema review.

Future JSON rules:

1. Any rationale JSON field must be optional and backward-compatible.
2. Existing fields such as `reason_for_priority`, `why_it_matters`, `ranking_explanation`, score, label, and priority must remain unchanged.
3. Rationale fields must not change ranking, taxonomy, source selection, or inclusion.
4. Schema tests must cover old JSON payloads and new JSON payloads.
