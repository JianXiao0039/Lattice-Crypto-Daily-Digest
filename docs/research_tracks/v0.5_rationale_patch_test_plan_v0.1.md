# v0.5 Rationale Patch Test Plan v0.1

Implemented in Phase 13I:

- Abstract-rich LWE/PQC paper produces structured rationale.
- Title-only paper is marked `title_only`.
- Keyword-only match does not produce fake method or contribution.
- Conclusion is used only when present.
- Missing full paper text creates TODO_VERIFY caveat.
- Weak FHE application paper is marked peripheral/temporary, not core.
- No manual annotation fields are required.
- Python 3.11 compatibility is checked by `py_compile`.

Future integration tests:

- Daily JSON includes rationale fields without dropping existing fields.
- Daily Markdown includes rationale without changing record ordering.
- Weekly synthesis can reuse rationale without inventing missing evidence.
- Monthly synthesis, if implemented, preserves coverage/source-starved caveats.
