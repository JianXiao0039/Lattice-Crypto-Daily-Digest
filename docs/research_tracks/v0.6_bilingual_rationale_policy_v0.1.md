# v0.6 Bilingual Rationale Policy v0.1

Status: `bilingual_rationale_polished`.

Top-paper rationale should be bilingual when it helps the user decide whether to 精读, 扫读, 暂存, or 忽略 a paper.

Apply full bilingual rationale to:

- A-class papers;
- Must Read papers;
- top Weekly papers;
- top Monthly papers;
- high-priority reading queue papers when exported fields support it.

Use compact Chinese-only rationale for lower-priority B/C records when full bilingual output would make generated documents too long.

Evidence rules:

- Abstract-derived summaries may only use the abstract.
- Conclusion-derived summaries may only appear when a conclusion field exists.
- Title-only and metadata-only records must remain conservative.
- Keyword hits may support radar relevance but cannot be the entire rationale.
- Proof, security, parameter, benchmark, and implementation claims remain TODO_VERIFY unless explicitly present in local fields.

This policy changes explanation rendering only. It does not change ranking, thresholds, taxonomy, source fetching, source health, or paper inclusion.
