# v0.5 arXiv Rate Limit Policy v0.1

Status: `compliant_low_load_policy_ready`.

arXiv HTTP 429 must be classified as `rate_limited`, not as a generic fetch failure.

## Required Behavior

- Honor `Retry-After` when present.
- Use low-load manual probes for diagnosis.
- Avoid tight retry loops.
- Reduce query volume or grouping when repeated 429s occur.
- Generate source-starved artifacts when arXiv is red and other sources also fail.

## Forbidden Behavior

- Proxy rotation.
- Fake User-Agent rotation.
- CAPTCHA or access-control bypass.
- Ignoring `Retry-After`.
- Aggressive repeated retries.
