# v0.5 Shadow Classifier Non-Integration Policy v0.1

The shadow classifier must not:

- be imported by production modules;
- run during Daily or Weekly workflows;
- rewrite `data/`, `digests/`, `handoffs/`, reading queues, or scores;
- alter production taxonomy or track assignments;
- become scheduled or background execution;
- promote rules without human annotation and controlled evaluation.

Any future integration proposal requires explicit review, repository-scoped tests, human-gold metrics, CI evidence, and a separate implementation phase.
