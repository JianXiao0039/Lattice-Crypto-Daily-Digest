# v0.6 Obsidian Deep Reading Scaffold Policy v0.1

Status: `obsidian_scaffold_improved`.

Paper notes should default to `status: unread` and should not claim that the user has read the paper.

Required sections:

- Radar Decision;
- 中文推荐理由;
- English Rationale;
- Direction Mapping;
- Deep Reading Checklist;
- Claim Verification;
- Research Use;
- Links.

The note must preserve TODO_VERIFY caveats and evidence basis. Title-only or metadata-only records must remain conservative.

Exports stay inside the repository unless the user explicitly authorizes a real vault path. This phase does not write to `D:\ResearchOS`.
