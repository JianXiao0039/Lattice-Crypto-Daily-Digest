# v0.4 Post-Tag Observation Plan v0.1

## Entry Condition

Begin release observation only after the exact release commit has green Ubuntu and Windows CI and consistent release metadata.

## Daily Observation

- inspect generated Markdown and JSON;
- record source green/yellow/red counts;
- label 0 records plus all-red sources as source-starved;
- record IACR latest status and count;
- record Semantic Scholar enrichment status without exposing credentials;
- verify no private workspace writes;
- verify final Git status.

## Weekly Observation

- confirm weekly input exists;
- run the handoff generator;
- record candidate and exclusion counts;
- preserve source-starved warnings;
- verify non-claims and TODO_VERIFY fields;
- avoid interpreting empty handoff as no relevant literature.

## Escalation

Use Full Manual Quality Run once, manually, when CI regresses, all sources remain red, required artifacts disappear, or release hygiene fails.
