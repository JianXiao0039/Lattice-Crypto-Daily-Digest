# v0.5 Durable Artifact Policy v0.1

Status: `durable_artifact_verifier_ready`.

Daily, Weekly, and Monthly radar outputs are durable only when the Markdown/JSON pair exists, the JSON parses, the Markdown is non-empty, and source-health/source-starved evidence is represented honestly.

## Daily Requirements

- `digests/YYYY-MM-DD.md` exists.
- `data/YYYY-MM-DD.json` exists.
- JSON is parseable.
- Markdown has a date or heading.
- Source-health data exists or is explicitly marked missing.
- Source-starved status is explicit when records are zero and all sources are red.

## Weekly Requirements

- `digests/weekly/YYYY-Www.md` exists.
- `data/weekly/YYYY-Www.json` exists.
- JSON is parseable.
- Source-health summary exists.
- Input date coverage exists.

## Monthly Requirements

- `digests/monthly/YYYY-MM.md` exists.
- `data/monthly/YYYY-MM.json` exists.
- JSON is parseable.
- Input daily files and missing days are listed.
- Monthly source-health summary includes source-starved status.
