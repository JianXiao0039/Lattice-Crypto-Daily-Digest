# v0.5 Source Health Recovery Plan v0.1

Status: `source_health_diagnostics_ready`.

Scope: manual diagnostics and conservative recovery only. This plan does not change ranking, taxonomy, query expansion, source selection, or Daily/Weekly/Monthly scheduling.

## Diagnostic Classes

- DNS failure: name resolution or local resolver problem.
- TLS failure: certificate validation, handshake, or local CA/proxy problem.
- HTTP 429: source rate limit; honor `Retry-After` and reduce load.
- HTTP 401/403: authentication, key, permission, or access policy issue.
- HTTP 5xx: source-side server error.
- Timeout: request exceeded safe timeout.
- Parser failure: reachable response could not be parsed.
- Empty response: valid response with zero records; not the same as network failure.
- Missing key: optional enrichment key absent; radar must degrade gracefully.
- IACR failed-attempt guard/cache hit: latest feed state must be reported explicitly.

## Recovery Policy

1. Run `python scripts/probe_source_health.py --low-load`.
2. Inspect per-source status and retryability.
3. For rate limits, wait and retry manually after the advertised window.
4. For TLS failures, inspect local CA/proxy configuration; do not disable SSL verification by default.
5. For empty responses, verify query/date filters before treating the day as quiet.
6. For source-starved runs, generate artifacts with explicit caveats.
7. Verify durable outputs with `python scripts/verify_durable_artifacts.py --date <YYYY-MM-DD> --week <YYYY-Www> --month <YYYY-MM>`.
