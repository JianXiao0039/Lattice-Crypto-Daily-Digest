# v0.5 Offline Rule Candidates v0.1

These are shadow-mode design candidates only.

- Require a lattice/PQC anchor before generic commitment, signature, privacy, AI, or implementation terms can assign a track.
- Treat ring/linkable signatures without lattice/SIS/LWE evidence as ambiguous.
- Treat generic ML model names such as Falcon as irrelevant unless cryptographic context is explicit.
- Keep classical lattice attacks in Track C even when no machine learning is present; record AI assistance separately.
- Prevent Track D from absorbing generic non-lattice PQC, generic FHE applications, and unrelated systems papers.

No candidate is approved for production.
