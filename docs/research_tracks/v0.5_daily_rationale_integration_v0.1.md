Status: `daily_rationale_integrated_with_limits`.

Integration point:

- `src/lattice_digest/digest.py`
- Daily Markdown now calls `build_recommendation_rationale(record)` inside `_basic_paper_lines`.

Rendered fields:

- `Paper problem`
- `Method / construction / attack / implementation`
- `Main contribution`
- `Radar relevance`
- `Why read / skim / track / ignore`
- `Evidence basis`
- `TODO_VERIFY`

Scope:

- Markdown explanation only.
- Existing paper ordering is unchanged.
- Existing relevance scores are unchanged.
- Existing relevance labels are unchanged.
- Existing reading-priority calculations are unchanged.
- Existing JSON schema is unchanged in this phase.

Evidence behavior:

- Abstract-rich papers receive abstract-derived summaries.
- Conclusion-derived support is used only when a conclusion field exists.
- Title-only or keyword-only records stay low-confidence.
- Weak FHE application/system papers are marked peripheral/temporary rather than core lattice papers.
