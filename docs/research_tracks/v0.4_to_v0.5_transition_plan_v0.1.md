# v0.4 to v0.5 Transition Plan v0.1

## Release Repair First

1. Commit the missing path-normalization implementation with its test.
2. Update package version, CHANGELOG, and release notes consistently.
3. Push the release candidate commit and require green Ubuntu and Windows CI.
4. Resolve the existing `v0.4.0` tag through an explicit release-history decision.
5. Run project tests, release hygiene, and diff hygiene again.

## v0.5 Scope After Repair

- improve source-health stability metrics;
- refine track-based weekly handoff quality;
- preserve manual recovery and source-starved guards;
- improve Module-SIS chameleon-hash research usefulness;
- keep private application and ResearchArtifacts boundaries unchanged.

## Non-Goals

- no scheduled retry service;
- no private PhD tooling;
- no automatic tag movement;
- no ranking or taxonomy overhaul without a separate phase.
