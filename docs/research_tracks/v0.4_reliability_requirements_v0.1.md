# v0.4 Reliability Requirements v0.1

- Missing daily artifacts must be reported explicitly.
- `0 records + all-red sources` must be labeled source-starved.
- Empty digest under source-starved input is not a successful paper-discovery result.
- Latest daily artifact alone must not hide source-starved history inside the same audit window.
- Daily dashboard must report record count, source counts, IACR latest status, Semantic Scholar status, and manual recovery need.
- Validation must include tests, release hygiene, `git diff --check`, and `git status -sb`.
