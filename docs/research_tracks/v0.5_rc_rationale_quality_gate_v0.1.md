# v0.5 RC Rationale Quality Gate v0.1

Status: `rationale_quality_gate_passed_with_limits`.

## Gate

Recommendation rationale must not be keyword-only. It should use available title, abstract, conclusion, metadata, repository notes, and source-health context.

## Current Evidence

- Monthly synthesis includes `Problem`, `Method`, `Contribution`, `Radar relevance`, `Reading action`, `Evidence basis`, and `TODO_VERIFY`.
- Reading queue records include rationale-derived fields and reading actions.
- Obsidian notes include structured recommendation and paper-work sections.
- Daily and Weekly selected artifacts still emphasize source-health summaries and lattice/PQC anchor evidence; they are usable but less complete than Monthly/queue/Obsidian.

## Decision

The gate passes with limits because the richer rationale layer exists and is exported, but Daily/Weekly Markdown remains compact.
