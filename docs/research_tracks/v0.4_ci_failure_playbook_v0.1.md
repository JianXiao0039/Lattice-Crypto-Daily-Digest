# v0.4 CI Failure Playbook v0.1

## Classify First

1. Confirm push and remote tree state.
2. Identify the failing workflow, job, and step.
3. Separate test failures from release hygiene and diff hygiene failures.
4. Compare Windows and Ubuntu matrix results.

## Inspect Safely

- use `gh run list` and `gh run view --log` only when authenticated;
- otherwise inspect public Actions metadata and state that detailed logs are unavailable;
- never invent a failing test name;
- reproduce in a clean Linux/Python 3.11 environment when possible.

## Test Failure Rules

- run repository-scoped tests;
- preserve the same Python version and dependency set as CI;
- use a full Git clone when doctor/release hygiene needs `.git`;
- check path separators, case sensitivity, permissions, temp cleanup, and shell differences;
- do not weaken or skip tests merely to obtain green CI.

## Release Hygiene Failure Rules

- inspect version consistency;
- inspect release document and CHANGELOG presence;
- inspect staged/generated artifact policy;
- keep private files and secrets out of diagnostics.

## Completion

A local fix is not CI green. Push the fix only after review, then verify the exact commit in both matrix jobs.
