# v0.5 Shadow-Mode Noninterference Policy v0.1

Shadow mode must not write `papers.db`, `data/`, `digests/`, weekly data, handoffs, reading queues, or production source/package files.

It must not be imported or invoked by Daily, Weekly, source-health, ranking, or release workflows. It must not install ARS, schedule itself, create a service, or access private workspaces.

Tests hash production artifact families before and after an isolated run, reject production output directories, scan workflows and the production package for entrypoint references, and verify the absence of production/ARS runtime imports.
