# v0.3.0 - Research Workflow Stabilization Release

## 中文说明

## 1. 发布定位

v0.3.0 是稳定化发布，不是新功能扩张发布。它把 v0.2.0 的文献库互操作能力继续向“可手动触发、低负载、可审计的研究工作流”收敛。

本版本不改变 fetcher、ranking、source health 语义、日报 8 节结构、weekly synthesis 聚合规则、reading queue 状态语义、Obsidian scaffold 模板或 research progress 生成语义。

## 2. 稳定内容

v0.3.0 稳定以下能力：

- CI / release hygiene checks。
- Source Health Ledger 持久化审计。
- Ranking Explainability。
- Research-oriented daily digest sections。
- Weekly Research Synthesis。
- Research Artifact Export Pack。
- Reading Queue Workflow + Review Status Tracker。
- Obsidian Paper Note Scaffold。
- Advisor Update + Research Progress Log。
- Workflow Command Center。
- Manual Low-Load Workflow Profiles。

## 3. Workflow Command Center

命令中心提供：

```powershell
python -m lattice_digest.workflow daily
python -m lattice_digest.workflow weekly
python -m lattice_digest.workflow full
python -m lattice_digest.workflow status
python -m lattice_digest.workflow doctor
```

安全默认值：

- `daily`、`weekly`、`full` 默认 dry-run。
- 写文件必须显式传入 `--execute`。
- Obsidian note scaffold 即使在 `weekly --execute` 下也默认 dry-run，除非传入 `--generate-notes`。
- `status` 和 `doctor` 只读。

## 4. Manual Low-Load Profile

v0.3.0 明确支持手动低负载运行：

```powershell
python -m lattice_digest.workflow weekly --low-load
python -m lattice_digest.workflow daily --low-load
python -m lattice_digest.workflow full --low-load
```

`--low-load` 只在用户显式请求时生效。普通模式语义不变。

## 5. 不添加本地自动调度

本版本不添加：

- Windows Task Scheduler 集成。
- cron job。
- startup task。
- background service。
- background daemon。
- automatic scheduled runs。
- 自动本地定时运行。

所有 workflow command center 命令都由用户手动触发。

## 6. 推荐验证命令

Windows PowerShell：

```powershell
Set-Location "D:\Code\CodexProjects\lattice-crypto-daily-digest"
python -m pytest tests --basetemp=.pytest_tmp
python scripts\check_release_hygiene.py
git diff --check

python -m lattice_digest.workflow weekly --low-load --skip-hygiene
```

## 7. 生成物策略

不要把以下运行产物混入功能或 release commit：

- `exports/`
- `exports/workflow-runs/`
- `exports/research-progress/`
- `exports/obsidian-paper-notes/`
- `audits/`
- `state/reading-queue.json`
- `data/*.json`
- `data/weekly/`
- `digests/*.md`
- `digests/weekly/`
- `papers.db`
- `.env`

## 8. 已知限制

- 外部 API 仍可能限流、超时或返回不完整 metadata。
- GitHub Actions provisional 日报仍可能弱于本地 authoritative backfill。
- 低负载 profile 不是调度器；它只降低手动运行时的默认压力。
- 命令中心只是编排层，不替代人工阅读、数学核验或实验复现。
- 本项目是科研 triage 与 workflow automation，不是正式 bibliography authority database。

## 9. v0.4.0 候选方向

- 更强的 dashboard / local API。
- 更细的 reading queue review workflow。
- Zotero integration 的进一步 dry-run / explicit-confirmation 流程。
- 更完善的 source reliability analytics。

## English Summary

v0.3.0 is a stabilization release for the research workflow layer. It packages CI and release hygiene, persistent source health ledgers, ranking explainability, research-oriented digest sections, weekly synthesis, research artifact exports, reading queue tracking, Obsidian note scaffolding, advisor progress logs, the workflow command center, and manual low-load profiles.

This release does not change fetcher behavior, ranking thresholds, source health semantics, daily digest section generation, weekly synthesis aggregation, reading queue status semantics, Obsidian templates, or research progress semantics. It also does not add local scheduled automation: no Windows Task Scheduler integration, no cron jobs, no startup tasks, no background services, and no automatic local scheduled runs.
