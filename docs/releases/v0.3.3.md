# v0.3.3 - IACR Recovery and Report Quality Maintenance Release

## 中文说明

## 1. 发布定位

v0.3.3 是 v0.3.2 之后的 maintenance release。它聚焦 IACR 手动恢复、latest/RSS 可观测性、跨源覆盖审计、可选 Semantic Scholar 元数据增强，以及 daily / weekly / research report 的研究可读性打磨。

本版本不添加自动调度，不改变 fetcher、ranking thresholds、taxonomy semantics、section classifier、query expansion、negative keywords、source health semantics、daily / weekly workflow、reading queue、Obsidian scaffold、research progress、Zotero export 或 release hygiene 的业务语义。

## 2. 本版本包含

- IACR failed attempt manual retry recovery：失败的 IACR 尝试不会永久阻塞后续手动恢复重试。
- IACR latest/RSS source recovery and latest-feed observability：IACR latest/RSS 路径可被枚举，并通过 `latest_feed_*` 字段暴露 latest feed 状态。
- Cross-source latest/query/enrichment audit：记录 IACR ePrint、arXiv、Semantic Scholar 及其他配置源的 latest-feed / query-only / enrichment-only 边界。
- Source query coverage audit：帮助维护者确认 source query 覆盖范围与降级状态。
- Optional Semantic Scholar metadata enrichment：Semantic Scholar 只作为可选 metadata enrichment，不是 primary source，也不是 ranking authority。
- Semantic Scholar key safety through `SEMANTIC_SCHOLAR_API_KEY`：只记录环境变量名，不在代码、文档、日志或测试中写入真实 API key。
- Research report quality polish：报告更清楚地区分 paper facts、ranking explanation、source health uncertainty、research relevance 和 action recommendation。
- Daily report lattice/PQC anchor evidence for high-priority papers：高优先级论文会显示 `lattice/PQC anchor evidence`，例如 ML-KEM、Kyber、LWE、Module-SIS、BKZ 等锚点证据。
- Manual quality-first workflow remains supported：用户仍可手动触发高质量运行；low-load / no-network / dry-run 是低负载、诊断或降压 fallback modes，不替代质量优先的手动运行。

## 3. 安全边界

No scheduled automation is added. 本版本不添加、不启用、不修改：

- Windows Task Scheduler。
- cron。
- startup task。
- background service。
- background daemon。
- watcher。
- automatic scheduled local run。

No ranking threshold changes. 本版本不改变 A/B/C/D 分数阈值、ranking weights 或 ranking labels。

No taxonomy semantic changes. 本版本不改变 taxonomy config、negative keyword semantics、query expansion semantics 或 section classifier semantics。

Semantic Scholar citation metadata is advisory only. `citationCount`、`influentialCitationCount`、venue、year、CorpusId、DOI、openAccessPdf 等只作为人工 triage 的参考信息，不能覆盖 A/B/C/D 排名。

## English Summary

v0.3.3 is a maintenance release after v0.3.2. It focuses on IACR failed attempt manual retry recovery, IACR latest/RSS source recovery, latest-feed observability, cross-source latest/query/enrichment audit, source query coverage audit, optional Semantic Scholar metadata enrichment, and research report quality polish.

Semantic Scholar remains optional enrichment only. The API key is read from `SEMANTIC_SCHOLAR_API_KEY`, and citation metadata is advisory only; it does not override ranking.

High-priority daily reports now surface `lattice/PQC anchor evidence` so A-level items are easier to inspect for ML-KEM, Kyber, LWE, Module-SIS, BKZ, and related anchors.

No scheduled automation is added: no Windows Task Scheduler integration, no cron jobs, no startup tasks, no background services, no watchers, and no automatic scheduled local runs. No ranking threshold changes and no taxonomy semantic changes are included.
