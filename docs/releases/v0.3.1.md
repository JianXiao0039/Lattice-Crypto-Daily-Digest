# v0.3.1 - Manual Operations Patch Release

## 中文说明

## 1. 发布定位

v0.3.1 是 v0.3.0 之后的 patch release。它不添加新产品功能，不改变 fetcher、ranking、source health、section classifier、daily digest、weekly synthesis、reading queue、Obsidian scaffold、research progress 或 workflow command center 的语义。

本版本聚焦两个目标：

- 用 deterministic fixture-based E2E acceptance suite 固化手动低负载研究工作流。
- 补齐面向操作者的 manual operations、recovery、artifact retention 和 troubleshooting 文档。

## 2. 本版本包含

- Phase 9A：End-to-End Workflow Acceptance Suite。
- Phase 9A hotfix：历史 release 测试归档化，不再要求当前版本等于旧版本。
- Phase 9B：Manual Operations Runbook。
- Recovery Playbook。
- Artifact Retention Policy。
- Troubleshooting docs。

## 3. 手动工作流边界

v0.3.1 继续保持 manual-only workflow：

- 默认 dry-run。
- 写文件需要显式 `--execute`。
- 低负载运行需要显式 `--low-load`。
- no-network/offline 需要显式 `--no-network` 或 `--offline`。

本版本不添加：

- Windows Task Scheduler。
- cron job。
- startup task。
- background service。
- watcher。
- automatic scheduled local run。

## 4. E2E acceptance 覆盖

E2E suite 使用 `tests/fixtures/e2e/` 中的确定性 fixture，覆盖：

- weekly synthesis from daily JSON。
- research artifact export from weekly JSON。
- reading queue import 保留人工状态。
- Obsidian scaffold dry-run 与生成。
- research progress advisor update / verification backlog。
- workflow weekly dry-run 与 `--execute` 临时目录写入。
- `Falcon-X` 不误入 PQC Falcon。
- `Practical Anonymous Two-Party GBDT` 不误入 AI4Lattice / LWE / SIS / PQC。
- 不创建 scheduled automation 文件。

## 5. 新增操作者文档

- `docs/manual-operations-runbook.md`
- `docs/recovery-playbook.md`
- `docs/artifact-retention-policy.md`
- `docs/troubleshooting.md`
- `docs/e2e-acceptance-suite.md`

这些文档说明 dry-run default、low-load mode、offline/no-network usage、哪些命令写文件、哪些命令只读、generated artifacts 不应提交、reading queue backup、`papers.db` recovery、Windows SQLite file lock、`tzdata` / `ZoneInfo` 和 CI failure triage。

## 8. 已知限制

- E2E suite 是 fixture-based acceptance，不替代真实网络源质量评估。
- 外部 API 仍可能限流、超时或返回不完整 metadata。
- 本地 authoritative backfill 仍需要用户手动运行。
- 本项目仍是科研 triage 与 workflow automation，不是正式 bibliography authority database。

## English Summary

v0.3.1 is a patch release after v0.3.0. It adds deterministic E2E acceptance tests for the manual low-load research workflow, fixes stale archival release tests, and adds operator-facing runbook, recovery, artifact retention, and troubleshooting documentation.

This release does not change fetchers, ranking, source health semantics, section classification, daily digest generation, weekly synthesis, reading queue behavior, Obsidian scaffold behavior, research progress behavior, or workflow command center semantics. It also does not add scheduled automation: no Windows Task Scheduler, no cron jobs, no startup tasks, no background services, no watchers, and no automatic local scheduled runs.
