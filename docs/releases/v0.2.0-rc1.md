# v0.2.0-rc1 - Library Interoperability Release Candidate

## 中文说明

## 1. 发布定位

v0.2.0-rc1 是 Library Interoperability Release Candidate，不是 stable v0.2.0。它用于验证 Phase 7A 的 Stable Library Export Layer 是否已经足够可用，能够把每日 digest JSON 转换成 Zotero / Obsidian / BibTeX / RIS / CSL JSON 可消费的稳定文献库格式。

这个版本不包含 Zotero XPI plugin，也不声称 Zotero 导入质量已经完成完整人工审计。它是走向 v0.2.0 stable 的候选版本。

## 2. 本版本解决的问题

v0.1.0 解决的是“能部署、能运行、能生成格密码科研日报”。v0.2.0-rc1 解决的是“已有 digest 数据能与文献库工具互操作”：

- 从 `data/*.json` 中读取 digest records。
- 统一转换为 stable library item。
- 生成 Zotero-ready CSL JSON。
- 同时提供 BibTeX / RIS 兼容导出。
- 生成 Zotero tags 与 import report。
- 引入深度格密码 taxonomy，便于后续 Zotero collection、Obsidian 双链、dashboard 和插件复用。

## 3. 核心能力

- Stable library item schema：`schemas/library-item.schema.json`。
- Deep lattice cryptography taxonomy：`src/lattice_digest/library_taxonomy.py`。
- Zotero-ready export：文件式导出，不是 XPI 插件。
- CSL JSON：`library-items.csl.json`。
- BibTeX：`library-items.bib`。
- RIS：`library-items.ris`。
- Zotero tags：`zotero-tags.json`。
- Import report：`import-report.md`。
- PowerShell export script：`scripts/export_library.ps1`。

## 4. 标签体系覆盖

v0.2.0-rc1 的 taxonomy 覆盖：

- lattice problems：LWE、RLWE、MLWE、SIS、Module-SIS、Ring-SIS、NTRU、SVP、CVP、BDD、q-ary lattice、module lattice。
- lattice reduction：LLL、BKZ、G6K、fplll、sieving、enumeration、pruning、GSA、lattice estimator。
- lattice cryptanalysis：primal attack、dual attack、hybrid attack、secret recovery、distinguisher、failure oracle、sample selection。
- PQC schemes：ML-KEM、Kyber、ML-DSA、Dilithium、Falcon、FN-DSA、FrodoKEM、Saber、NIST PQC。
- primitives：commitment、Module-SIS commitment、chameleon hash、FHE、CKKS、BFV、BGV、TFHE、ZK-friendly primitives。
- implementation security：constant-time、side-channel、fault attack、masking、NTT、production audit、benchmark、artifact evaluation。
- AI4Lattice：Transformer LWE、Swin-guided coordinate selection、negative-cyclic modeling、hybrid ranking、neural lattice reduction、learned pruning、BKZ parameter prediction。
- research workflow tags：Must Read、Recommended Read、Skim、Archive、Paper Idea、Experiment Candidate、TODO_VERIFY。

## 6. Zotero 当前支持方式

当前不是 Zotero XPI 插件。推荐的 Zotero workflow 是文件式手动导入：

1. 运行 `scripts\export_library.ps1`。
2. 生成 CSL JSON / BibTeX / RIS。
3. 在 Zotero 中使用 `File -> Import`。
4. 检查 authors / year / DOI / URL / tags / notes。
5. 下一阶段 Phase 7B 做导入质量审计和 taxonomy confusion audit。

## English Summary

v0.2.0-rc1 is a Library Interoperability Release Candidate, not the stable v0.2.0 release. It packages the Stable Library Export Layer so existing digest JSON can be transformed into stable library items, Zotero-ready CSL JSON, BibTeX, RIS, Zotero tags, and import reports.

This release validates the file-based interoperability path before building a dashboard, local API, or Zotero XPI plugin. The Zotero plugin is not included. The current Zotero workflow is manual file import: export CSL JSON / BibTeX / RIS, import into Zotero, and manually inspect authors, dates, DOI, URL, tags, and notes.

The release includes a deep deterministic taxonomy for lattice cryptography topics such as LWE/RLWE/MLWE, SIS/Module-SIS/Ring-SIS, lattice reduction, lattice cryptanalysis, PQC schemes, lattice primitives, implementation security, FHE, ZK-friendly post-quantum privacy, and AI4Lattice. Full taxonomy quality auditing and Zotero manual import validation are planned for Phase 7B before v0.2.0 stable.
