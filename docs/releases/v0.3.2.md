# v0.3.2 - Documentation and Maintenance Release

## 中文说明

## 1. 发布定位

v0.3.2 是 v0.3.1 之后的 documentation and maintenance release。它不添加新产品功能，不改变 fetcher、ranking、source health、section classifier、daily digest、weekly synthesis、reading queue、Obsidian scaffold、research progress、Zotero export、release hygiene 或 workflow command center 的语义。

本版本聚焦四个目标：

- 把 manual low-load research workflow 的文档入口收束到 `docs/index.md`。
- 把 README 改成更清晰的 safe manual quickstart。
- 用 command safety matrix 明确 read-only / write-file / network / low-load 边界。
- 整理 one-week manual pilot docs 与 pilot feedback triage docs，为后续手动试运行反馈分类做准备。

## 2. 本版本包含

- Phase 9C：maintenance cleanup / warning reduction where applicable。
- Phase 9D：one-week manual pilot acceptance docs。
- Phase 9E：pilot feedback triage docs、summary template、fix prioritization rules。
- Phase 9J：documentation polish、`docs/index.md` documentation map、README safe manual quickstart、command safety matrix。
- Phase 9K：v0.3.2 release notes and version metadata。

## 3. 手动工作流边界

v0.3.2 继续保持 manual-only usage：

- workflow command center 保持 manual-only workflow。
- `daily`、`weekly`、`full` 保持 dry-run default。
- 写文件仍需要显式 `--execute`。
- low-load mode 仍需要显式 `--low-load`。
- no-network/offline usage 仍需要显式 `--no-network` 或 `--offline`，并在支持的 workflow 中跳过 network fetch。
- generated artifacts must not be committed by default。
- reading queue manual statuses / local state should be preserved。

No scheduled automation is added. 本版本不添加：

- Windows Task Scheduler integration。
- cron job。
- startup task。
- background service。
- background daemon。
- watcher。
- automatic scheduled local run。

## 7. 已知限制

- 文档抛光不替代真实 pilot feedback。
- 外部 API 仍可能限流、超时或返回不完整 metadata。
- 本地 authoritative backfill 仍需要用户手动运行。
- 本项目仍是科研 triage 与 workflow automation，不是正式 bibliography authority database。

## English Summary

v0.3.2 is a documentation and maintenance release after v0.3.1. It adds no product features and changes no workflow behavior.

This release packages documentation polish, the `docs/index.md` documentation map, the README safe manual quickstart, the command safety matrix, one-week manual pilot docs, pilot feedback triage docs, and maintenance cleanup / warning reduction where applicable.

The workflow remains manual-only. `daily`, `weekly`, and `full` keep dry-run default behavior; writes still require explicit `--execute`; low-load mode still requires explicit `--low-load`; no-network/offline usage still requires explicit `--no-network` or `--offline` where supported.

No scheduled automation is added: no Windows Task Scheduler integration, no cron jobs, no startup tasks, no background services, no background daemons, no watchers, and no automatic scheduled local runs.
